# 作物知识库一期说明

## 本期目标

第一版先完成以下闭环：

- 维护作物品类、作物品种、生长阶段
- 为作物/品种/阶段配置推荐目标区间
- 在区域管理中绑定结构化作物信息
- 在独立页面按区域生成推荐建议，并对比当前监测值

## 页面入口

- `智能分析 -> 作物知识库`
- 路由：`/ai/crop-knowledge`

## 后端接口

- `GET /api/v1/crop-knowledge/area-options`
- `GET /api/v1/crop-knowledge/options`
- `GET/POST/PUT/DELETE /api/v1/crop-knowledge/species`
- `GET/POST/PUT/DELETE /api/v1/crop-knowledge/varieties`
- `GET/POST/PUT/DELETE /api/v1/crop-knowledge/stages`
- `GET/POST/PUT/DELETE /api/v1/crop-knowledge/targets`
- `GET /api/v1/crop-knowledge/recommendation?areaId=...`

## 数据模型

新增表：

- `agri_crop_species`
- `agri_crop_varieties`
- `agri_crop_growth_stages`
- `agri_crop_target_profiles`

区域表新增字段：

- `biz_areas.crop_species_id`
- `biz_areas.crop_variety_id`
- `biz_areas.crop_stage_id`

## 当前推荐逻辑

1. 先读取区域绑定的作物品类/品种/生长阶段。
2. 查询匹配的目标画像。
3. 品种专属目标优先；若不存在，则回退到作物品类 + 阶段的通用目标。
4. 读取该区域每个指标最近一条监测值。
5. 输出：
   - 推荐区间
   - 最佳值
   - 偏低 / 偏高 / 正常 / 未接入
   - 建议摘要

## 一期默认内置样例

- 作物：`番茄`
- 品种：`樱桃番茄`
- 生长阶段：`开花坐果期`
- 推荐指标：
  - 温度
  - 湿度
  - EC
  - pH
  - CO2
  - 光照
  - 土壤氮
  - 土壤磷
  - 土壤钾

## 当前边界

- 当前推荐目标主要是结构化知识库驱动，不依赖爬虫。
- 当前值来自已有传感器接入；未接入的指标会标记为 `未接入`。
- 还没有按作物自动生成规则模板、灌溉策略或 AI 解释链路。

## 下一步建议

- 给区域绑定“栽培方式 / 种植日期 / 当前阶段切换”
- 增加更多作物和品种模板
- 把推荐目标联动到规则引擎、告警阈值和 AI 建议
