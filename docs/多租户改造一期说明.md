# 多租户改造一期说明

## 本期目标

本期只完成多租户改造的基础底座，不直接切换为真正的多租户隔离运行模式。

当前完成的内容：

- 新增 `sys_tenants` 租户主表
- 核心主表补充 `tenant_id`
- 初始化脚本补充默认租户与演示数据归属
- 后端鉴权上下文预埋 `tenant`
- 用户、角色、系统配置、设备接入凭证等核心写入路径支持自动写入当前租户

## 本期覆盖的核心表

- `sys_tenants`
- `sys_users`
- `sys_roles`
- `sys_data_scopes`
- `biz_areas`
- `iot_gateways`
- `iot_sensors`
- `iot_actuators`
- `rule_definitions`
- `ops_alerts`
- `ops_control_commands`
- `ai_tasks`
- `ai_reports`
- `sys_configs`
- `iot_media_nodes`
- `iot_cameras`

## 本期刻意没有做的内容

- 还没有把所有查询统一改成 `tenant_id` 强制过滤
- 还没有把所有唯一索引改成“租户内唯一”
- 还没有新增租户管理页面
- 还没有把历史派生表、日志表、会话表全部补齐租户字段
- 还没有切换成真正的多租户 SaaS 运行模式

## 兼容策略

为了兼容现有单租户系统，本期采用以下策略：

- `tenant_id` 先允许为空
- 未完成迁移前，后端会回退到隐式默认租户上下文
- 迁移脚本会把现有主数据全部回填到 `default` 租户

## 上线顺序建议

1. 执行 [100_multi_tenant_foundation.sql](/Users/mac/Documents/New%20project/agri-platform/sql/100_multi_tenant_foundation.sql)
2. 确认 `sys_tenants` 和核心表 `tenant_id` 已创建
3. 检查默认租户是否已生成，现有数据是否已回填
4. 再部署带 tenant 上下文的新后端代码
5. 第二阶段再开始做查询隔离和租户管理页面
