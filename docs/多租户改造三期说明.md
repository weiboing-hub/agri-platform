# 多租户改造三期说明

第三期目标是把系统配置正式切到租户维度，同时保持现网默认租户和现有设备接入不受影响。

## 本期范围

- `sys_configs` 从“全局唯一”调整为“租户内唯一”
- AI 服务配置读取按当前租户优先
- 通知通道配置读取按当前租户优先
- 设备接入凭证读取按当前租户优先
- 系统主题读取支持“当前租户 -> 默认租户 -> 全局”的回退链路

## 数据库变更

- 新增迁移脚本：`sql/110_multi_tenant_config_scope.sql`
- 将 `sys_configs.tenant_id IS NULL` 的历史记录回填到默认租户
- 将唯一索引从：
  - `uk_sys_configs_group_key (config_group, config_key)`
- 调整为：
  - `uk_sys_configs_tenant_group_key (tenant_id, config_group, config_key)`

## 行为变化

### 系统配置读取

配置读取优先级变为：

1. 当前租户配置
2. 默认租户配置
3. 全局配置（仅兼容旧数据）

### 设备接入凭证

- `系统管理 -> 设备接入凭证` 会按当前租户读取和轮换 token
- 设备接口校验仍兼容环境变量 token，避免现网设备在迁移期中断

### AI 与通知测试

- 系统设置页中的 AI/通知联调会读取当前租户的配置

## 本期未覆盖

- 用户名、角色编码、设备编码等唯一索引仍是全局模型
- 设备接入 token 仍是“按租户一条配置”，还不是“每设备单独凭证”
- 通知记录表本身还没有补 `tenant_id`

## 上线建议

1. 先执行 `sql/110_multi_tenant_config_scope.sql`
2. 再部署后端代码
3. 重启 `agri-admin-api`
4. 验证：
   - `系统设置 -> 登录安全`
   - `系统设置 -> 设备接入凭证`
   - `系统设置 -> 联调测试`
   - 首页主题与系统主题读取
