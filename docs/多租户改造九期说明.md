# 多租户改造九期说明

本期目标是把“租户管理”从主档维护推进到“可交付初始化”：

- 每个租户可维护独立的能力开关和资源配额
- 每个租户可查看、显式生成和轮换自己的设备接入令牌
- 每个租户可一键初始化默认角色、默认数据范围和首个租户管理员

## 本期范围

### 1. 租户运行配置

新增两组租户级配置：

- `tenant_features`
  - `enable_ai`
  - `enable_media`
  - `enable_openclaw`
  - `enable_alert_notifications`
- `tenant_limits`
  - `max_users`
  - `max_gateways`
  - `max_cameras`
  - `max_ai_tasks_per_day`

这些配置写入 `sys_configs`，并按租户维度隔离。

### 2. 租户设备接入凭证

新增租户级设备凭证工作台：

- 查看当前生效的设备 token 掩码
- 判断是“租户专属 token”还是“继承默认 token”
- 查看明文 token
- 重新生成租户专属 token

设备鉴权仍保持兼容：

- 如果租户存在显式 token，则优先使用租户 token
- 如果租户未显式配置，则仍可回退到默认租户/环境变量 token

### 3. 租户初始化

新增租户初始化能力：

- 以默认租户为模板复制默认角色
- 复制默认数据范围
- 复制角色与权限关系
- 复制角色与数据范围关系
- 自动补齐租户级运行配置
- 自动生成租户级设备 token
- 创建首个租户管理员并绑定 `super_admin`

## 数据库调整

### `sys_data_scopes`

原先 `scope_code` 为全局唯一，不利于多个租户共享同样的数据范围模板（如 `all_areas`、`owner_areas`）。

本期改为：

- 删除全局唯一索引 `uk_sys_data_scopes_scope_code`
- 新增租户内唯一索引 `uk_sys_data_scopes_tenant_scope_code (tenant_id, scope_code)`

## 兼容性说明

- 现网默认租户继续可用
- 未显式配置租户 token 的租户，仍可继承默认 token
- 设备控制和设备上报既有逻辑不需要立即切换
