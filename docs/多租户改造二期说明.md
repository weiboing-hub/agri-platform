# 多租户改造二期说明

本期目标：在不改变现网默认租户运行方式的前提下，补上查询侧 tenant 隔离，并让核心新增写入自动带 tenant_id。

本期已覆盖：
- 工作台、实时监控、设备中心、控制中心、告警中心、AI、系统管理的核心查询
- 区域、网关、传感器、执行器、媒体节点、摄像头、规则、AI 任务/报告、控制命令、IoT 自动建档的 tenant_id 写入
- AI 全局范围查询改为“租户内全局”

本期未完成：
- 唯一索引改为“租户内唯一”
- sys_configs 唯一键改为 tenant 维度
- device credential 改为真正的租户级凭证
- 审计日志、通知、部分派生明细表的 tenant 字段补齐

上线后预期：
- 现网默认租户功能不变
- 新增数据会自动归属默认租户
- 后续再新增租户时，核心读路径已有 tenant 隔离基础
