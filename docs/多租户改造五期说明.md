# 多租户改造五期说明

第五期开始处理“低风险的租户内唯一约束”，目标是在不碰登录名和设备编码的前提下，把区域、规则、角色这三类内部编码切到租户范围。

## 本期范围

- `sys_roles.role_code` 改成租户内唯一
- `biz_areas.area_code` 改成租户内唯一
- `rule_definitions.rule_code` 改成租户内唯一
- 启动时默认管理员挂载 `super_admin` 角色时，改为只查默认租户
- 设备自动接入默认区域时，改为在当前租户内按 `AREA-AUTO-DEFAULT` 查找
- 种子脚本中按 `role_code / area_code / rule_code` 查找的语句，全部补上默认租户或关联租户条件

## 数据库变更

- 新增迁移脚本：`sql/130_multi_tenant_low_risk_unique_keys.sql`

索引调整如下：

- `sys_roles`
  - 从 `uk_sys_roles_role_code (role_code)`
  - 调整为 `uk_sys_roles_tenant_role_code (tenant_id, role_code)`

- `biz_areas`
  - 从 `uk_biz_areas_area_code (area_code)`
  - 调整为 `uk_biz_areas_tenant_area_code (tenant_id, area_code)`

- `rule_definitions`
  - 从 `uk_rule_definitions_rule_code (rule_code)`
  - 调整为 `uk_rule_definitions_tenant_rule_code (tenant_id, rule_code)`

## 代码变更

- `lib/bootstrap.js`
  - 绑定 `super_admin` 角色时，只查默认租户

- `routes/iot.js`
  - `ensureDefaultArea()` 改成租户内查询默认接入区

- `sql/020_seed_rbac.sql`
  - 默认角色权限和数据范围赋权改成默认租户限定

- `sql/030_seed_demo_data.sql`
  - 演示区域、规则、AI 数据的按编码查找改成默认租户或关联租户限定

## 本期价值

- 多个租户可以复用相同的 `area_code`
- 多个租户可以复用相同的 `rule_code`
- 多个租户可以复用相同的业务角色编码，例如 `platform_admin / supervisor`

## 本期未覆盖

- `username` 仍然是全局唯一
- `gateway_code / sensor_code / actuator_code / camera_code` 仍然是全局唯一
- `scope_code` 仍然是全局唯一

这些对象后续要单独评估，因为它们会影响登录、设备上报和接入识别，风险明显更高。
