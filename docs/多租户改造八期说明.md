# 多租户改造八期说明

第八期开始把“租户”从数据库里的基础表，推进成真正可在后台管理的对象。本期重点是租户主档管理，不涉及套餐计费、租户管理员初始化和业务数据初始化向导。

## 本期范围

- 新增租户管理页面
  - 租户列表
  - 新增租户
  - 编辑租户
  - 启用 / 停用
  - 切换默认租户
- 新增租户管理接口
  - `GET /api/v1/system/tenants`
  - `POST /api/v1/system/tenants`
  - `PUT /api/v1/system/tenants/:id`
- 新增租户管理权限
  - `tenant:manage`
- 登录页继续复用公开租户选项接口
  - 启用租户才会出现在登录页下拉中

## 权限边界

- 租户管理入口需要 `tenant:manage`
- 后端额外要求“默认租户下的 `super_admin`”才能真正执行租户管理
- 即使后续有人误把 `tenant:manage` 配给普通角色，也不能越权管理租户
- 非平台超管在角色权限配置页看不到 `tenant:manage`

## 默认租户规则

- 默认租户必须保持 `enabled`
- 不能直接把当前默认租户取消默认或停用
- 正确做法是先把另一个启用租户设为默认，系统会自动撤掉旧默认

## 本期价值

- 多租户从“底层隔离”推进到“后台可运营”
- 登录入口、默认租户和租户主档开始形成完整闭环
- 后续可以在此基础上继续做：
  - 租户管理员初始化
  - 套餐与配额
  - 租户级设备凭证
  - 租户级对象存储与文件隔离

## 本期未覆盖

- `username` 仍然保持全局唯一
- 还没有做租户管理员初始化向导
- 还没有做套餐、计费、设备数限制
- 还没有做租户级独立域名 / 子域名自动识别
