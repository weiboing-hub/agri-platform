# 多租户改造十一期说明

## 本期目标

在已有“租户主档、查询隔离、租户级配置、租户级设备凭证、媒体衍生表租户化”的基础上，继续补齐 SaaS V1 的运营底座：

- 新增租户套餐与订阅模型
- 支持查看和维护租户当前订阅
- 将运行配额从“仅配置”升级为“关键写接口强校验”

## 本期改动

### 1. 套餐与订阅底座

新增表：

- `sys_tenant_plans`
- `sys_tenant_subscriptions`

内置套餐：

- `trial`
- `standard`
- `enterprise`
- `internal`

系统会为未绑定订阅的现有租户自动补齐默认套餐：

- `trial` 租户 -> `trial`
- `internal` 租户 -> `internal`
- 其他租户 -> `standard`

### 2. 生效配额解析

新增后端帮助层：

- `admin-api/src/lib/tenant-entitlements.js`

生效规则按优先级叠加：

1. 系统默认值
2. 套餐默认值
3. 订阅覆盖值
4. 租户运行配置覆盖值

当前生效额度包含：

- `max_users`
- `max_gateways`
- `max_cameras`
- `max_ai_tasks_per_day`

当前生效能力开关包含：

- `enable_ai`
- `enable_media`
- `enable_openclaw`
- `enable_alert_notifications`

### 3. 关键写接口强校验

新增后端强校验：

- 创建用户时校验 `max_users`
- 创建网关时校验 `max_gateways`
- 创建摄像头时校验 `max_cameras`
- 创建 AI 任务/报告时校验 `max_ai_tasks_per_day`

同时新增能力开关校验：

- 未启用 `enable_ai` 时，禁止创建和重试 AI 任务
- 未启用 `enable_media` 时，禁止创建媒体节点、摄像头、抓图和 FTP 抓图接收

### 4. 租户工作台升级

`租户管理` 页面新增：

- 套餐与订阅区
- 订阅状态维护
- 套餐切换
- 到期时间维护
- 生效配额与当前使用量概览

## 新增接口

- `GET /api/v1/system/tenant-plans`
- `GET /api/v1/system/tenants/:id/subscription`
- `PUT /api/v1/system/tenants/:id/subscription`

## 兼容性说明

- 现网默认租户会自动获得默认订阅，不影响现有登录和设备上报
- 旧的租户运行配置仍然生效，但现在会作为“最高优先级覆盖层”
- 设备侧 token、控制链路、媒体抓图链路保持兼容

## 下一步建议

如果继续推进 SaaS 商业化，建议下一阶段进入：

1. 套餐/订阅到期控制
2. 租户配额超限后的更细粒度提示与引导
3. 对象存储路径、导出文件、媒体目录按 `tenant/` 归档
