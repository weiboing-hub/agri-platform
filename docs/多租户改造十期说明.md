# 多租户改造十期说明

本期目标是把媒体衍生数据从“依赖摄像头主表间接隔离”升级为“记录本身就带租户归属”。

## 本期范围

- `iot_camera_capture_jobs`
- `iot_camera_snapshots`
- `iot_camera_recordings`
- `ai_image_analysis_results`

以上表全部补充 `tenant_id`，并完成历史数据回填。

## 设计动机

此前媒体相关列表已经能通过摄像头主表做租户过滤，但衍生记录自身没有 `tenant_id`，会带来几个问题：

- 后续对象存储路径和离线任务难以直接按租户分桶
- 做跨表统计时必须依赖额外 join
- 做数据归档、导出、清理时不够直观
- 图像分析结果不能直接表达所属租户

本期改造后，媒体衍生链路会形成：

- 摄像头 `iot_cameras.tenant_id`
- 抓图任务 `iot_camera_capture_jobs.tenant_id`
- 抓图结果 `iot_camera_snapshots.tenant_id`
- 录像索引 `iot_camera_recordings.tenant_id`
- 图像分析结果 `ai_image_analysis_results.tenant_id`
