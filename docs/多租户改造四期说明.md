# 多租户改造四期说明

第四期先处理“派生表租户归属”，目标是让通知记录和操作日志不再依赖关联表间接判断租户。

## 本期范围

- `ops_notifications` 增加 `tenant_id`
- `sys_operation_logs` 增加 `tenant_id`
- 通知列表、通知重发、通知联调统计改为按 `ops_notifications.tenant_id` 过滤
- 操作日志查询改为按 `sys_operation_logs.tenant_id` 过滤
- 审计日志写入时自动记录当前租户

## 数据库变更

- 新增迁移脚本：`sql/120_multi_tenant_derived_tables.sql`
- 历史通知记录回填规则：
  1. 优先取 `ops_alerts.tenant_id`
  2. 其次取 `ai_tasks.tenant_id`
  3. 最后回退到默认租户
- 历史操作日志回填规则：
  1. 优先取操作人 `sys_users.tenant_id`
  2. 否则回退到默认租户

## 代码变更

- `lib/audit.js`：操作日志写入时带 `tenant_id`
- `routes/system.js`：系统通知联调统计、审计日志查询按租户过滤
- `routes/alerts.js`：通知列表和通知重发按租户过滤

## 本期价值

- 通知中心不再依赖告警表才能判断租户归属
- 审计日志终于具备租户级隔离能力
- 后续做租户管理员视角时，通知与审计数据可以独立分页和统计

## 本期未覆盖

- 用户名、角色编码、区域编码、设备编码等全局唯一索引还未切到租户内唯一
- 通知编号 `notification_no` 仍然保持全局唯一
- 操作日志编号 `log_no` 仍然保持全局唯一
