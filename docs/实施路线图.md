## 建议执行顺序

### 第一步：先做后端底座

- 建库建表
- 登录鉴权
- 用户、角色、权限、数据范围
- 区域、网关、传感器、执行器 CRUD

### 第二步：打通主业务链路

- 实时监控
- 历史分析
- 告警中心
- 手动控制
- 控制回执

### 第三步：补齐平台能力

- 规则引擎
- 自动控制策略
- Device Shadow
- AI 任务队列与报告

### 第四步：补摄像头与媒体链路

- 摄像头设备模型
- 媒体服务层
- 图片 / 录像存储
- 图像 AI 结果回写

## 为什么现在最应该先做数据库和 API

- 页面已经定义得很细，继续写页面文档收益不高
- 如果表结构没定，页面字段一定会来回改
- 如果 API 没定，前端和后端会并行失焦
- 权限、告警、控制、AI 这些都是强后端驱动能力
