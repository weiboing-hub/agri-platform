# 数据库设计说明

## 1. 设计目标

这套库结构服务于以下主链路：

```text
区域/设备管理 → 传感器采集 → 历史数据 → 规则判断 → 告警流转 → 控制执行 → AI 诊断/报告 → 审计
```

数据库设计重点：

- 把“设备、传感器、执行器、影子状态”拆开
- 把“告警主状态”和“告警流转记录”拆开
- 把“控制请求”和“执行回执”拆开
- 权限采用“角色权限 + 用户覆盖 + 数据范围”
- 历史数据显式记录数据来源、补传标记、时间可信度

## 2. 表分层

### 系统层

- `sys_users`：用户表
- `sys_roles`：角色表
- `sys_permissions`：权限点表
- `sys_user_roles`：用户角色关联
- `sys_role_permissions`：角色默认权限
- `sys_user_permission_overrides`：用户权限覆盖
- `sys_data_scopes`：数据范围定义
- `sys_role_data_scopes`：角色默认数据范围
- `sys_user_data_scope_overrides`：用户数据范围覆盖
- `sys_operation_logs`：操作日志
- `sys_configs`：系统配置

### 业务对象层

- `biz_areas`：区域表
- `iot_gateways`：网关表
- `iot_sensors`：传感器表
- `iot_actuators`：执行器表
- `iot_device_shadow`：设备影子状态表

### 数据与运行层

- `iot_sensor_readings`：历史采集数据
- `iot_gateway_backfill_batches`：补传批次记录

### 规则、告警、控制层

- `rule_definitions`：规则定义
- `ops_alerts`：告警主表
- `ops_alert_transitions`：告警流转表
- `ops_control_commands`：控制指令表
- `ops_control_executions`：控制执行结果表
- `ops_notifications`：通知记录表

### AI 层

- `ai_tasks`：AI 任务队列表
- `ai_reports`：AI 报告表

## 3. 关键设计点

### 3.1 历史数据可信度

`iot_sensor_readings` 强制保留以下字段：

- `data_source`
- `is_backfilled`
- `collected_at`
- `received_at`
- `clock_synced`
- `time_uncertainty_ms`
- `time_quality`
- `delay_ms`

这样前端历史分析页才能明确区分：

- 实时数据
- 补传数据
- 时间高可信 / 中可信 / 低可信

### 3.2 Device Shadow

`iot_device_shadow` 单独建表，不和执行器主表混写。

原因：

- 期望状态与实际状态是两个独立信号
- 需要追踪状态偏差持续时长
- 后端控制逻辑和 UI 呈现都要围绕 Shadow 展示

### 3.3 告警状态机

`ops_alerts.status` 建议固定支持：

- `pending`
- `acknowledged`
- `in_progress`
- `on_hold`
- `ignored`
- `closed`
- `reopened`

所有流转动作写入 `ops_alert_transitions`，不要只改主表状态。

### 3.4 控制闭环

控制链路拆为两层：

- `ops_control_commands`：发起的控制请求
- `ops_control_executions`：设备回执与执行结果

这样才能严格区分：

- 已发送
- 已收到
- 已执行
- 执行失败
- 执行超时

### 3.5 权限即时生效

后端鉴权推荐基于以下组合：

```text
角色默认权限
+ 用户权限覆盖
- 用户撤销权限
+ 角色默认数据范围
+ 用户数据范围覆盖
```

高危权限变更后，需要清理登录态或强制刷新 Token。

## 4. 索引建议

### 高频列表与检索

- 告警页：`ops_alerts(status, severity, triggered_at)`
- 实时监控：`iot_gateways(online_status, last_heartbeat_at)`
- 历史分析：`iot_sensor_readings(sensor_id, collected_at)`
- 设备列表：`iot_sensors(area_id, sensor_type)`、`iot_actuators(area_id, actuator_type)`
- 控制记录：`ops_control_commands(request_status, queued_at)`
- AI 队列：`ai_tasks(status, scheduled_at)`

### 后期可能要补的汇总表

当数据量上来后，可以新增：

- 日级环境统计表
- 区域实时快照表
- 告警统计汇总表
- 控制执行统计表

第一版先不做，先用主表 + 索引即可。
