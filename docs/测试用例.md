# 测试用例

## 1. 测试范围

本轮交付重点覆盖以下模块：

- 登录与权限初始化
- 区域 / 网关 / 传感器 / 执行器管理
- 手动控制与控制记录
- 告警中心与流转
- 规则引擎与自动控制策略
- AI 诊断 / AI 报告 / AI 任务队列
- 系统设置
- 设备影子状态
- 通知记录
- 指标字典
- 多指标总览图表
- ESP32 / 网关设备接入
- 媒体节点 / 摄像头管理
- 抓图任务 / 历史图片时间轴

## 2. 测试环境

- 前端：`admin-web`
- 后端：`admin-api`
- 数据库：`agri_iot_platform_dev`
- 默认管理员：
  - 用户名：`admin`
  - 密码：`Admin@123456`

## 3. 冒烟测试用例

### TC-001 登录成功

- 前置条件：数据库已执行初始化脚本
- 步骤：
  1. 打开登录页
  2. 输入 `admin / Admin@123456`
  3. 点击登录
- 预期结果：
  - 跳转到总览大屏
  - 本地会话写入 `accessToken`
  - 左侧菜单正常显示

### TC-002 总览与监控接口正常返回

- 步骤：
  1. 进入总览大屏
  2. 进入实时监控
  3. 进入历史分析
- 预期结果：
  - 页面无报错
  - 能看到演示数据
  - 时间字段可正常展示

### TC-003 执行器控制命令入队

- 步骤：
  1. 打开手动控制页
  2. 选择执行器
  3. 发起 `on` 或 `off`
- 预期结果：
  - 返回“控制命令已入队”
  - 控制记录页新增一条记录
  - 目标执行器 `shadow_status` 变为 `pending`

### TC-004 告警流转

- 步骤：
  1. 进入告警中心
  2. 选择一条告警
  3. 分别执行 `confirm / process / close`
- 预期结果：
  - 告警主状态更新
  - 流转记录新增
  - 操作日志新增对应审计记录

### TC-005 规则新建与编辑

- 步骤：
  1. 进入规则引擎
  2. 新建一条阈值规则
  3. 修改其启用状态或摘要字段
- 预期结果：
  - 列表能看到新规则
  - 详情页 JSON 正常展示
  - 审计日志新增 `rule_engine` 记录

### TC-006 AI 诊断生成

- 步骤：
  1. 进入 AI 诊断页
  2. 发起一次手动诊断
- 预期结果：
  - 生成一条 `ai_tasks`
  - 同时生成一条 `ai_reports`
  - 页面能看到诊断摘要、风险等级、Markdown 内容

### TC-007 AI 报告生成

- 步骤：
  1. 进入 AI 日报 / 周报页
  2. 选择 `daily` 或 `weekly`
  3. 点击立即生成
- 预期结果：
  - 任务创建成功
  - 报告列表新增记录
  - 详情中能看到版本号与报告内容

### TC-008 AI 任务重试

- 步骤：
  1. 进入 AI 任务队列
  2. 找到失败任务
  3. 点击重试
- 预期结果：
  - 任务状态更新为 `success`
  - 任务重试次数 +1
  - 新增关联报告

### TC-009 系统设置保存

- 步骤：
  1. 打开系统设置
  2. 修改一项参数，例如 `平台名称`
  3. 点击保存
- 预期结果：
  - 返回“系统设置已保存”
  - `sys_configs` 中对应记录更新
  - 操作日志新增 `system_settings` 记录

### TC-010 系统联调测试

- 步骤：
  1. 在系统设置页先配置 `ai_provider` 和 `notification_channel`
  2. 分别点击
     - 测试 AI 服务
     - 测试通知
     - 测试设备连接
- 预期结果：
  - 三个接口均返回结构化结果
  - AI 测试返回 `mode/model/baseUrl`
  - 通知测试返回 `provider/httpStatus`
  - 页面显示最近一次测试结果

### TC-011 设备影子状态查询

- 步骤：
  1. 打开设备影子状态页
  2. 按 `shadowStatus` 筛选
  3. 查看详情
- 预期结果：
  - 列表返回执行器影子状态
  - 详情可看到 `desired / reported / lastCommand / drift`

### TC-012 设备影子重同步

- 步骤：
  1. 在设备影子状态页点击重同步
- 预期结果：
  - 返回“已触发影子状态重同步”
  - 目标执行器 `shadow_status` 更新为 `pending`
  - 审计日志新增 `device_shadow` 记录

### TC-013 通知记录查询

- 步骤：
  1. 打开通知记录页
  2. 按渠道和状态筛选
  3. 查看详情
- 预期结果：
  - 能看到演示通知记录
  - 详情显示关联告警、响应信息、发送时间

### TC-014 通知重发

- 步骤：
  1. 在通知记录页选择失败记录
  2. 点击重发
- 预期结果：
  - 状态变为 `sent`
  - `retry_count` 增加
  - 审计日志新增 `notification_logs` 记录

### TC-015 指标字典新增与编辑

- 步骤：
  1. 打开“系统管理 -> 指标字典”
  2. 新增一条指标，填写编码、名称、单位、范围、图表颜色
  3. 再编辑该指标的名称或启用状态
- 预期结果：
  - 列表出现新指标
  - 详情页能看到范围、颜色和通道统计
  - 保存后接口返回成功

### TC-016 多指标总览图表展示

- 步骤：
  1. 打开“总览大屏”
  2. 查看多指标趋势卡片
  3. 点击不同指标卡片切换主图
- 预期结果：
  - 页面无空白和报错
  - 指标卡显示最新值、最小值、最大值、均值
  - 主图会切换到对应指标，补传占比柱正常显示

### TC-017 规则可视化编辑器

- 步骤：
  1. 打开“规则引擎”
  2. 在可视化模式下选择目标类型、具体目标、触发指标
  3. 添加告警 / 通知 / 控制 / AI 动作卡片
  4. 保存规则
- 预期结果：
  - 下拉指标来源于指标字典
  - 目标选择区、动作卡片区正常联动
  - 保存后列表和详情页都能看到中文摘要

### TC-018 设备上报接入

- 步骤：
  1. 调用 `POST /api/v1/iot/ingest`
  2. 使用当前 ESP32 兼容 payload 上报 `device_id / temp / hum / rssi`
  3. 再查看网关、传感器、实时监控页
- 预期结果：
  - 若设备未建档，系统自动创建默认接入区、网关、温度/湿度传感器和通道
  - `iot_sensor_readings` 成功新增两条记录
  - 网关变为 `online`
  - 传感器当前值与最后采集时间已更新

### TC-019 媒体节点新增与编辑

- 步骤：
  1. 打开“媒体节点管理”
  2. 新增一个媒体节点
  3. 再编辑其 HLS 地址或 FTP 根路径
- 预期结果：
  - 列表出现媒体节点
  - 详情区显示 API / RTMP / HLS / FTP 信息
  - 保存后无报错

### TC-020 摄像头新增与编辑

- 步骤：
  1. 打开“摄像头管理”
  2. 新增一个摄像头并绑定区域、网关、媒体节点
  3. 保存后再编辑源流地址
- 预期结果：
  - 列表出现摄像头
  - 详情区显示源流地址和安装位置
  - 主码流配置已落库

### TC-021 手动抓图

- 步骤：
  1. 打开“抓图任务”
  2. 选择摄像头
  3. 点击“立即抓图”
- 预期结果：
  - 新增一条成功状态的抓图任务
  - `iot_camera_snapshots` 新增一条 `manual` 图片记录
  - 历史图片时间轴可看到新增记录

### TC-022 FTP 抓图元数据接收

- 步骤：
  1. 打开“抓图任务”
  2. 在“模拟 FTP 接收”填写路径和时间
  3. 提交
- 预期结果：
  - 返回“FTP 抓图接收成功”
  - `iot_camera_snapshots` 新增一条 `ftp_upload` 记录
  - 时间轴按 `FTP 上传` 过滤可看到图片索引

### TC-023 历史图片时间轴筛选

- 步骤：
  1. 打开“历史图片时间轴”
  2. 分别按区域、摄像头、来源类型、时间范围筛选
- 预期结果：
  - 筛选结果与数据库记录一致
  - 详情区能看到 `filePath / ftpPath / capturedAt / receivedAt`
  - 页面不空白、不报错

## 4. 建议回归清单

- 每次改动权限模型后，回归：
  - 登录
  - 菜单可见性
  - 关键写操作按钮
- 每次改动控制链路后，回归：
  - 手动控制
  - 控制记录
  - 设备影子状态
- 每次改动 AI 模块后，回归：
  - 诊断生成
  - 报告生成
  - 任务重试
  - 系统设置中的 AI 检测
- 每次改动指标字典或规则编辑器后，回归：
  - 指标字典新增/编辑
  - 规则引擎可视化保存
  - 自动控制策略指标下拉
  - 总览多指标趋势
- 每次改动设备接入后，回归：
  - `/api/v1/iot/ingest`
  - `/api/soil/ingest`
  - 网关在线状态更新
  - 实时监控最新值展示
